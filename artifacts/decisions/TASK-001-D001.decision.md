## Issue

The `template/` directory in the repository was stored as an orphaned gitlink (a `160000 commit` tree entry pointing to commit `4756556a15d24c90e928ff9284a7c5ef6db736c4`) with no corresponding `.gitmodules` entry. This means:

1. The directory was neither a tracked regular directory nor a properly registered submodule.
2. Cloning the repo with standard `git clone` would produce an empty `template/` directory — the commit it referenced was unreachable from any remote.
3. The template sync protocol (CLAUDE.md, `docs/orchestration.md §9`) requires changes to workflow files to be copied into `template/` and committed. With a gitlink entry, `git add template/` was a no-op — changes written to the filesystem were silently ignored by git.

Without resolving this, the template sync protocol was structurally impossible to follow correctly.

## Options Considered

### Option A: Re-register as a proper git submodule
Register `template` as a submodule pointing to a separate repository. This requires a separate GitHub repository for the template, adding external dependency and ongoing sync overhead.

### Option B: Convert to a regular tracked directory (chosen)
Remove the orphaned gitlink entry (`git rm --cached template`) and re-add the directory as a regular tracked tree (`git add template/`). This keeps everything self-contained in a single repository and makes template sync work with standard git commands.

### Option C: Revert all template sync work to a separate PR
Revert the gitlink removal and submit the structural change as a dedicated PR before doing template sync.

## Chosen Option

**Option B**: Convert `template/` to a regular tracked directory within this repository.

## Reasoning

- The original gitlink was already broken — the referenced commit (`4756556a`) was unreachable from any remote. Any user cloning the repo would have obtained an empty `template/` directory regardless, so there is no regression.
- Option A adds external dependency that does not match the self-contained design of this repo.
- Option C is operationally blocked: if we defer the structural fix, the template sync required by CLAUDE.md is impossible to complete in any PR, and the first `docs/` change will create permanent drift between docs and template.
- No `.gitmodules` file exists in the repo, confirming there was never a properly registered submodule.

## Implications

- `template/` is now a regular subdirectory. Files in it are tracked and versioned normally.
- Future template syncs (per `docs/orchestration.md §9`) work with standard `cp` + `git add` + `git commit`.
- The gitlink-based workflow is permanently retired for this repo.
- Any contributor or automation that expected a submodule checkout at `template/` must now treat it as a regular directory.
