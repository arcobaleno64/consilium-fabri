## Objective
將 consilium-fabri 既有的 artifact-first multi-agent workflow 顯式重構為兩層架構：以 PDCA（Plan-Do-Check-Act）為「專案管理層」之循環骨幹，以 TAO（Thought-Action-Observation）/ ReAct 為「代理人執行層」之循環骨幹，使巨觀（跨任務）與微觀（單步驟）治理在文件、artifact schema、guard validator 三層皆有名實一致之承載。

## Background
- 現行 workflow（Intake → Research → Planning → Coding → Verification → Closure）已隱含完整 PDCA：plan(P) + premortem R1-R4 → code(D) → verify(C) + Build Guarantee → improvement(A) + Gate E + decision artifact，但無顯式命名。
- subagent（Implementer / Tester / Verifier）目前以「契約驅動」運轉（輸入規約 → 限定輸出 artifact），未要求顯式 Thought→Action→Observation 循環，亦無自省與重試明文。
- 兩者粒度不同（PDCA 巨觀跨任務、TAO/ReAct 微觀單步），定位互補而非競合，現行 artifact 規範可吸收兩層而不衝突。
- 使用者已裁定範圍為 Phase 1-5 全套（命名顯式化 + TAO 契約引入 + validator 擴充 + template sync + 既有 24 task 全數回填）；TAO Trace 中高 risk（risk ≥ 3）之 implementer/verifier 必填。
