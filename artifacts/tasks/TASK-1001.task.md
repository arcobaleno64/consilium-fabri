- **template sync**：所有 root 修改須同步至 [template/](template/) 對應路徑。
- **依賴 TASK-1000 完成**：Phase 3+4（涉 validator code 與 RACI Auditor）須等 TASK-1000 Phase 3+5 完成、guard validators 含 `--check-tao` 後啟動。Phase 1+2 文件部分可獨立先行。
- **Synthesizer 雪球效應防禦（C-1）**：[architecture-synthesizer/TEMPLATE.md](docs/templates/architecture-synthesizer/TEMPLATE.md) 之 Combine 階段（SECI 第三步）須具防雪球機制：(a) 每次 dispatch 前必讀 `architecture-synthesis-cache.md` 既有條目與相關 memory-bank 檔之最新版；(b) 每 cluster 須宣告 `Existing Cache Match` 欄位（none / superseded / duplicate-of:<section-id>）；(c) 每 Synthesis section 標示 `Reference Range: TASK-XXX..TASK-YYY`，避免 reference 範圍模糊重疊；(d) Claude 在 Combine 階段做去重 + 衝突解決，不直接 append 新 cluster；(e) 衝突須產 decision artifact，不得偷偷覆寫舊條目。
- **RACI Auditor 違規與豁免分離（C-2）**：[guard_contract_validator.py](artifacts/scripts/guard_contract_validator.py) 之 RACI Auditor 邏輯須將 **Violation 偵測** 與 **Waiver 套用** 顯式分離：(a) Violation 偵測為純函數，僅依 file path × agent identity × 既定 RACI 矩陣推算，獨立於任何 decision artifact；(b) validator output 永遠先列原始 violations 全集，再列 post-waiver 結果；(c) Waiver 須 ex-ante（decision artifact 之 `Last Updated` ≤ 違規 commit 時戳）；(d) retroactive waiver 須顯式標記 `waiver_type: retroactive`，並記錄人工 approver；(e) 任何 violation（即使 waived）必入 audit log [.github/memory-bank/raci-violations-log.md](.github/memory-bank/raci-violations-log.md)（append-only），不得抹除。

  - 必查：[artifacts/registry/risk_classification.csv](artifacts/registry/risk_classification.csv) 存在且 row 數 = TASK-XXX 任務數（24+）
  - 必查：24 既有 task 之 plan/code/verify artifact 已含 `PDCA Stage` metadata 欄

- **Audit-Driven Development（C-4，ADD）**：AC-10 之實作須採 test-first：
  - **先寫 failing unit test**：`artifacts/scripts/test_guard_units.py` 加至少 3 個測試 case：(a) 同一 file × agent 在無 decision artifact 下回傳 violation；(b) 同一 file × agent 在有 ex-ante decision 下 violation 不變、post-waiver 集合縮減；(c) `Waiver Type: pre-existing-dirty` 之 decision 不應對「task session 內 agent 行動所產生之 violation」生效（即 TASK-1000 之豁免類別不能擴張至 retroactive-violation 情境）。
  - **再寫 implementation**：通過上述 3 case 後方可合併。
  - **PR/commit 須含證據**：每次與 RACI Auditor 相關之 commit message **必須引述 [.github/memory-bank/raci-violations-log.md](.github/memory-bank/raci-violations-log.md) 之 diff 行範圍**（如 `Audit-log-delta: +12 lines (TASK-1001 violation #1-#3 + 1 ex-ante waiver applied)`），無此引述視為不完整 commit。

- **commit message 規範（C-5）**：Codex 啟動 Phase 3+4 時，每次與 audit log 相關之 commit 須含：(a) 觸發 commit 之 plan_item 編號；(b) raci-violations-log.md 之 diff 行範圍與條目計數；(c) 若涉 waiver，附 decision artifact 路徑與 Waiver Type。範例：`feat(raci-auditor): violation detection + ex-ante waiver. Audit-log-delta: +5 (TASK-1001 plan_item 6.2). Waiver: artifacts/decisions/TASK-1001.decision.md (Waiver Type: ex-ante).`
